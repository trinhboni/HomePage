manga rock

INTRODUCTION

Manga Rock is an online manga reader that is available for both IOS and Android. There are about 2 millions users on IOS before it hit Android. The App supports more than 7 languages ( English, Italian, Vietnamese, German, French, Spanish and Chinese ) and there are serveral manga sources for each languages with over 100,000 titles. Users are able to download manga and use it on the go.

UX PROBLEM

Many users has complained about the screen size not being compatible on some devices. Sometimes pictures become to zoomed out/in which makes it harder to read. Further, for most users, they felt very overwhelmed with the amount of information that displays on the home screen.

UI PROBLEM

Three UI problems that exist within the app include: 

“ I want to know the action is in progress”

“ I want the interaction to feel natural

“ I don’t want to make my screen a mess ”

THE GOAL

The main goal for the project is enhance the user experience by improving the app's UX and UI motion design to encourage more manga reader to use reading app. From my research, the app interface was very messy and all over places, it also does not have a loader. I decided to recreate most of the app's interface and added two short animations that could solve these UI problems.

PROCESS

[ PROCESS ]

uSER research

In order to have a better understanding about the app, I did some research. For my research, I wanted to know what other users think about the app and looked at the review sections, then I found multiple articles/ blogs of users who shared their experiences. Further, I downloaded and experienced the app myself to see what the app's strengths and weaknesses are.

For my primary research, I interview people who read manga and have used the app. Most of them have answered that they prefer to read it on a laptop or website, rather than using the manga reading app. This primary research has provided me with a lot of valuable information, I was able to find-out what problems existed within the app and could develop solutions for these problems.

From my interviewee I created 3 diverse personas.

| User that regularly uses the app.
| User that most likely going to use the app.
| User that least likely to use the app.

initial sketches

My notes and sketches while developing and re-designing the app's interface.
[ PROCESS ]

wire frames

Based on the user-flow, I have created wire frames and redesigned most of the app's interface. For the home screen, since most of the users felt very overwhelmed with all the manga's displays. I have decided to replace it with a search bar. Therefore, the user can search up the one that they want to read and go straight to the reading. In case the user does not know what to read and want to explore the library, I have placed "Latest Update" section at the bottom of the page for the user to look through. The section will change to "Recommended" if the user has created an account and already logged in.

Further, on the reading screen, it looked very messy because of all the icons that are randomly placed all over the screen, so I combined them into a "control centre" button. Whenever user press the button, all the setting features will pop-out which makes it more convenient and not messy.

[ PROCESS ]

mockups

Since the app's original colour scheme is white and blue I have kept the same branding but changed the interface design.

[ PROCESS ]

UI motion story boards ( sketches )

After all the process of redesign the app, I decided to make 2 short animations to fix all the UI problems that the app is experiencing.

The first scenario is showing the user is reading a manga. However, the user is not comfortable with vertical reading and scrolling, so user opens the settings to switch it to horizontal layout. The purpose of this animation is to enhance the user experience through natural feeling while interact with the app.


For the scenario for the second animation, since the app does not have a loader which makes the user felt hanging whenever they opened the app. So I have created a loader and the reason I animated a person walking while the app is loading because one of the app's feature is the ability to download manga and read it on the go [ Manga on the go, MangaRock!! ] .
